A broken program

Here's a program you might reasonably be expected to write in CS 106A, in which you implement a function, sum_pairs, that computes the sum of each pair of elements in a list.

The program on the left is a first attempt at solving the problem, but unfortunately isn't quite right. Upon running the program in the terminal, it crashes, producing the output in the panel above.

That error message looks scary, but it's actually Python's way of providing as much useful information as it can to help you diagnose and cure your program of its bugs. In this tutorial, we'll review how to read an error message like this: which parts are important and equally, which parts you can safely ignore. Once we've done that, we'll take a look at how we can use this feedback to adjust and eventually fix our program.
