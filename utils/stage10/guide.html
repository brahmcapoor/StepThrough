Debugging with print

You might be most familiar with print as a function that outputs information for a user to read, but it's also an instrumental tool in inspecting the state of your program and your variables as you debug a program.

As a reminder, our current task is to identify which of the indices i or i + 1 on line 14 might be causing an IndexError in our code. The problem that presented itself to us was that we weren't certain which iteration of the for loop on line 13 the error was actually happening in.

Thus, it might be helpful for us to see how many times the for loop is executed before the crash occurs. In addition, since we're interested in the values of i and i + 1 as the for loop progresses, we might want to see their values too.
